# Claude Agent Rules

## Code Standards

- Write production-quality code
- Follow best practices
- Use existing tools/libraries before implementing custom solutions
- Adhere to the current coding style unless explicitly asked otherwise
- Follow the existing project structure unless explicitly asked otherwise
- Avoid magic strings/values; use constants
- Keep code readable, succinct, and straightforward

## Documentation

- Keep documentation succinct and to the point
- Update relevant docs when adding/changing features

## Commit Messages

Format: `{category}: {description}`

Categories:
- `feat` - New features
- `fix` - Bug fixes
- `refactor` - Code refactoring
- `docs` - Documentation
- `test` - Tests
- `build` - Build system
- `chore` - Maintenance

Example: `feat: add pause directive feature`
